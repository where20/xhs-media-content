# 封面文案风格指南 · 前端转 AI 电商内容

> **垂类：** 前端转 AI 电商内容
> **平台：** 小红书
> **核心理念：** 封面 = 用户刷到时的 0.5 秒决定 = 3 秒传达主题

---

## 🎨 一、视觉系统（统一性 > 单张好看）

### 1. 配色（建立强记忆点）

| 角色 | 颜色 | HEX | 用在哪 |
|------|------|-----|--------|
| **主色** | 墨绿 | `#0f766e` | 大字 / 数字 / 边框 |
| **强调** | 小红书红 | `#ff2442` | 警示 / 数字 / CTA |
| **辅助** | 米白 | `#fafaf7` | 背景 |
| **文字** | 深灰 | `#1a1a1a` | 正文 |
| **暗色** | 黑色 | `#000` | 代码块 / 工具名 |

**主色建议：** **墨绿**（与小红书红形成强反差，又能传达"程序员/理性"的感觉，比"全红"显专业）

### 2. 字体

| 用途 | 字体 | 字号 |
|------|------|------|
| 大字钩子 | 思源黑体 Bold / 阿里巴巴普惠体 Heavy | 80-120pt |
| 副标题 | 思源黑体 Medium | 40-50pt |
| 小字补充 | 思源黑体 Regular | 20-28pt |
| 数字 | DIN Alternate Bold / 阿里巴巴普惠体 Heavy | 90-130pt |
| 代码 | JetBrains Mono / Fira Code | 24-32pt |

> 工具：Canva 模板（搜"小红书封面"）、稿定设计、Figma

### 3. 关键元素（必带 1-2 个）

- ✅ **对比符号**：✅ ❌ 🎯 →❌ A→B 数字对比 30min → 5min
- ✅ **真实截图**：AI 输出 / 代码块 / 数据图（必带，不带不专业）
- ✅ **小标签**：顶部右上角小字标"前端视角" / "AI实测" / "避坑"
- ✅ **页码感**：D1 / D7 / D14 / D30（让用户知道"这是个系列"）

## 📐 二、3 种核心版式（直接套）

### 版式 1：数字冲击型（适合：复盘 / 数据 / 时间对比）

```
┌──────────────────────────┐
│  30天                    │  ← 顶部小字 副标题
│  ━━━━━                   │
│                          │
│   30min                  │  ← 主大字（颜色=主色）
│     ↓                    │  ← 箭头
│   5min                   │  ← 大字（颜色=小红书红）
│                          │
│  AI 帮我做商品图片归档   │  ← 底部小字 说明
└──────────────────────────┘
```

**适用：** "30min → 5min"、"5000 块 → 800 块"、"100个组件 → 1个工作流"

---

### 版式 2：对比表格型（适合：横评 / 避坑 / A/B/C）

```
┌──────────────────────────┐
│  4 类 AI 文案翻车        │  ← 顶部标题
│                          │
│   ❌ 规格参数            │  ← 3 行红叉（颜色=小红书红）
│   ❌ 功效承诺            │
│   ❌ 适用人群            │
│   ❌ 价格活动            │
│                          │
│   ✓ 发布前必查清单       │  ← 底部小字（颜色=墨绿）
└──────────────────────────┘
```

**适用：** "8 个常见坑"、"4 类事实"、"3 款工具对比"、"5 种开头模板"

---

### 版式 3：流程图型（适合：教程 / 工作流 / SOP）

```
┌──────────────────────────┐
│  一个工作流跑通 30 天    │  ← 顶部标题
│                          │
│  [商品资料]              │  ← 输入框
│     ↓                    │
│  [AI 提示词]             │  ← 流程节点
│     ↓                    │
│  [人工核对]              │
│     ↓                    │
│  [可发布文案]            │  ← 输出框
│                          │
│  附：完整提示词在评论区  │  ← CTA 小字
└──────────────────────────┘
```

**适用：** "客服回复库搭建"、"素材归档流程"、"数据复盘 SOP"

## 🗣️ 三、文案句式（5 套直接套）

### 1. 数字+反差型（数据感强）
- 30天把"商品上架"从 2 天压到 4 小时
- AI 写 10 个版本不如我手写 1 个
- 100 个组件 vs 1 个工作流，前端转电商的思维转变

### 2. 反常识+转折型（情绪感强）
- 写 100 个组件不如写 1 个工作流
- AI 不能跑通的 3 类电商任务（诚实记录）
- 我用 Git 的方式管理商品文案

### 3. 时间承诺型（立人设用）
- 32岁前端，决定公开 30 天 AI 电商实验
- 副业第 90 天复盘：赚了 3000 块，我学到了 3 件事
- 第一周复盘：什么有效、什么无效、下周测什么

### 4. 痛点+方案型（干货感强）
- 商品图片命名为什么这么乱？我用 JS 写了个规则引擎
- AI 文案翻车的不是文风，是这 4 类事实
- 客服回复库从 0 搭建：AI 整理 200 条对话

### 5. 身份+视角型（差异化用）
- 前端做了 5 年，我转行做电商的第一天
- 程序员做电商 1 年，我为什么放弃"全自动"想法
- 副业做电商，1 个月踩的 8 个坑（含代码）

## ✅ 四、封面自检清单（发布前 30 秒）

发布前用这张清单扫一眼：

- [ ] **3 米外能看清主标题**？（手机刷到时占屏 1/3）
- [ ] **主标题 ≤ 12 字**？（超了要拆成 2 行）
- [ ] **主色对比够强**？（主色 vs 背景 = 强对比）
- [ ] **有真实截图/数据**？（不全是装饰）
- [ ] **有对比符号**？（✅❌ / 数字 / 箭头 / D系列）
- [ ] **小标签位置一致**？（固定在顶部/右上）
- [ ] **不要小字过多**？（≤ 3 行小字）
- [ ] **不要全文字**？（必须有图/色块/视觉锚点）

## ❌ 五、5 个反面教材（千万别这么干）

### 反面 1：堆文字
```
我做电商5年了，这5年我学到了很多，今天分享一下我的经验。
在做电商的过程中，我尝试了很多方法，包括AI工具、SOP、自动化等等。
今天我就把这些经验分享给大家，希望对大家有帮助。
```
→ **没用。** 用户 0.5 秒就划走了。

### 反面 2：色块全开
```
粉色背景 + 黄色大字 + 蓝色边框 + 红色 emoji
```
→ **视觉噪音。** 反而让用户看不出重点。

### 反面 3：摆拍感太重
```
精致的办公室、咖啡、电脑摆拍，没有文字
```
→ **没钩子。** 用户不知道这篇在讲什么。

### 反面 4：标题党
```
震惊！90% 的电商人都不知道的 5 件事！！
```
→ **失去信任。** 一次标题党 = 长期低质量标签。

### 反面 5：全英文标题
```
AI Workflow Optimization for E-commerce
```
→ **小红书是中文场。** 除非个别单词（如 AI / API）否则都用中文。

## 🎁 六、3 套现成封面模板（直接套用）

### 模板 A：复盘型

```

工具：Canva 搜"小红书封面 极简"
```

## 📐 七、尺寸 & 规范

| 元素 | 规范 |
|------|------|
| **图片尺寸** | 1242 × 1660 px（小红书 3:4 首选）或 1080×1440 |
| **文件大小** | ≤ 5MB |
| **格式** | PNG（推荐）/ JPEG |
| **文字安全区** | 上下左右各留 60px（避免被裁切） |
| **主字号** | ≥ 80pt（保证手机可读） |
| **对比度** | 主文字 vs 背景 ≥ 4.5:1（WCAG AA 标准） |

## 💡 九、最后 1 个原则

> **一致性 > 单张好看。**
>
> 30 天用同一套配色 + 同一套字体 + 同一套版式。
> 用户刷到第 5 篇时会想"这又是我关注的那个号" = **系列感 = 关注欲**。
